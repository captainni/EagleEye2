# 政策分析详细参考标准

本文档提供政策分析技能的详细分析标准、判定规则和注意事项。

## 为什么输出格式如此重要？

⚠️ **关键原因**：
1. **系统自动解析** - 输出将被后端自动解析为 JSON 对象并存入数据库
2. **格式错误会导致失败** - 任何额外的文字、markdown 标记都会导致解析失败
3. **中文引号问题** - 中文引号 `"` 和 `'` 不是有效的 JSON 字符，必须转义
4. **高亮匹配依赖** - keyPoints 字段内容必须与原文精确匹配才能实现高亮显示

## 一、字段详细说明

### 1. policyType（政策类型）

**判定标准**：

| 类型 | 说明 | 典型来源 |
|------|------|----------|
| **监管政策** | 规范性文件、强制性规定 | 央行、银保监会、证监会、金融监管总局 |
| **行业指引** | 指导性文件、行业标准 | 协会、行业组织、监管机构 |
| **通知公告** | 一般性通知、公告 | 各监管部门、地方政府 |
| **其他** | 不符合以上分类 | 新闻报道、解读文章等 |

### 2. importance（重要程度）

| 级别 | 说明 | 判断标准 |
|------|------|----------|
| **高** | 涉及强制性合规要求、影响核心业务、违规后果严重 | 强制性规范、处罚条例、准入要求 |
| **中** | 涉及重要业务但非强制、或影响范围有限 | 指导性文件、鼓励性政策、过渡期安排 |
| **低** | 参考性信息、影响有限 | 一般通知、参考案例、行业动态 |

### 3. relevance（与我方产品的相关度）

**有产品信息时**：
- **高**：政策内容直接涉及我方核心业务或主要产品
- **中**：政策内容与我方有一定关联，但不是核心影响
- **低**：政策内容与我方关联度较低

**无产品信息时**：
- 默认为"中"

### 4. areas（领域标签）
- 提取 2-5 个相关领域关键词
- 常见标签：数据安全、个人信息保护、风险管理、跨境支付、金融科技、反洗钱、消费者权益保护、资本充足率、流动性风险、信贷政策等
- 标签应简洁、准确、便于分类

### 5. summary（摘要）
- 长度：200-300 字
- 内容：概括政策背景、主要内容、实施要求
- 风格：简洁明了，避免复制粘贴大段原文

### 6. keyPoints（关键条款 - ⚠️ 重要）
- 提取 3-5 条需要特别注意的具体条款
- **必须是原文语句，不得臆造或改写**
- 这些要点将用于在原文中**高亮显示**
- 每条要点应该是完整的句子或短语，便于精确匹配
- 选择对业务有价值、需要重点关注的内容

### 7. impactAnalysis（影响分析）
- 分析政策对银行业/保险业的整体影响
- 评估合规成本或业务机会
- 识别潜在风险点
- 视角：行业整体、监管趋势

### 8. suggestions（可执行建议）
- 提出 2-3 条可执行、可落地的建议
- 每条建议需说明理由和依据
- 建议应具体明确，避免泛泛而谈
- **聚焦于"我方应该如何应对"**

---

## 二、注意事项

### 关键规则
1. ✅ **keyPoints 必须是原文语句** - 用于精确匹配和高亮，这是核心功能
2. ✅ **所有分析必须基于文章内容** - 不得臆造信息
3. ✅ **建议要具有可操作性** - 避免空泛的建议
4. ✅ **impactAnalysis 要客观全面** - 分析行业影响和监管趋势
5. ✅ **suggestions 要聚焦行动** - 回答"我们应该如何应对"

### 常见错误
- ❌ keyPoints 使用概括性文字而非原文语句
- ❌ 添加文章中没有的信息
- ❌ 建议过于宽泛（如"加强关注"）
- ❌ 忽视具体业务场景，泛泛而谈

---

## 三、示例对比

### 优秀的 keyPoints（原文语句）
```
"金融机构应当建立数据分类分级管理制度"
"建立数据安全技术防护体系，保护个人信息"
"制定数据安全事件应急预案"
```

### 不好的 keyPoints（概括性文字）
```
"建立数据管理制度"  // 太宽泛，无法精确匹配
"加强安全防护"  // 不够具体
"完善应急预案"  // 原文中没有
```

## 四、快速参考

### 政策类型快速判断
- 看到"央行/银保监会/证监会发布" → 监管政策
- 看到"指导意见/指引/指南" → 行业指引
- 看到"通知/公告" → 通知公告
- 看到"解读/分析/报道" → 其他

### 重要性快速判断
- 有"必须/应当/强制/严禁" → 高
- 有"鼓励/引导/建议/支持" → 中
- 一般性通知/参考信息 → 低

### 相关度快速判断
- 直接涉及核心业务 → 高
- 相关业务领域 → 中
- 无直接关联 → 低
